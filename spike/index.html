Monday Apps Framework — SSE / External-Connect Spike

Phase 1.5B0 — Validates whether a Monday board-view iframe can open external HTTPS connections (including EventSource / SSE) to a non-Monday origin. If all three tests succeed, the v3 embedded-view architecture (engine → SSE → Marey renderer) is viable.

Test 1a — External SSE (Wikimedia recent changes stream)

Opens an EventSource to stream.wikimedia.org, Wikipedia's production SSE feed. Reliable real-time data, definitely up.

Test 1b — External SSE (sse.dev/test, redundancy)

Second external SSE endpoint as a cross-check. If Wikimedia works but this doesn't, CSP is not the issue.
IDLE

Test 2 — Same-origin fetch (baseline)

Plain fetch() against the iframe's own URL. Should always work. Sanity check that scripts run at all.
IDLE

Test 3 — Monday SDK context handshake

Loads monday-sdk-js and calls monday.get('context'). Confirms the iframe is actually running inside a Monday board view (not just embedded raw).

Interpreting results: All three OPEN/OK = full green. Test 1 ERROR + Tests 2/3 OK = Monday CSP is blocking external connect-src; we'd need to either proxy SSE through Monday's domain or pivot to polling. Tests 2/3 ERROR = iframe not loading scripts correctly (separate concern, possibly CSP script-src).
